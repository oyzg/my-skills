# Grill

Interview the user relentlessly about every aspect of this plan until reaching shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one. For each question, provide your recommended answer.

Ask the questions one at a time.

If a question can be answered by exploring the codebase, explore the codebase instead.

Never assume. If anything is ambiguous — scope, behavior, data model, user, error handling, permissions, non-functional requirements — ask explicitly. Do not silently default to what seems reasonable. Surface every assumption as a question so the user can confirm or correct it.
