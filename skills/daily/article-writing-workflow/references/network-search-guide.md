## 概览
本文档提供系统化的网络搜索方法，用于在文章创作过程中获取真实、准确、全面的信息。通过科学的搜索策略和多维度验证，避免 AI 幻觉问题。

## 搜索策略设计

### 1. 关键词设计
**原则**：从宽泛到精准，逐步细化

**步骤**：
- 第 1 轮：核心关键词（宽泛）
  - 例如："GPT-5"、"OpenAI"、"AI 写作"

- 第 2 轮：组合关键词（增加限定）
  - 例如："GPT-5 发布时间"、"OpenAI 官方公告"
  - 例如："AI 写作 工具 对比"、"GPT-5 vs GPT-4"

- 第 3 轮：精确关键词（目标明确）
  - 例如："GPT-5 参数量 多少"
  - 例如："OpenAI 2024年3月 发布会"

### 2. 搜索源选择
**优先级顺序**：
1. 官方来源（公司官网、官方博客、官方 Twitter/X）
2. 权威媒体（36氪、虎嗅、极客公园、TechCrunch）
3. 专业报告（艾瑞咨询、易观分析、IDC）
4. 用户反馈（知乎、小红书、微博）
5. 社区讨论（GitHub、Reddit、Stack Overflow）

### 3. 搜索时间范围
**根据需求设置**：
- 最新动态：最近 1-3 个月
- 中期趋势：最近 6-12 个月
- 长期背景：无时间限制

## 搜索维度

### 维度 1：官方资料
**目标**：获取第一手、最权威的信息

**搜索策略**：
```
关键词模式：
- "[公司名] 官方 发布"
- "[产品名] 官方 博客"
- "[CEO姓名] 演讲 采访"
- "[公司名] 财报 数据"

搜索示例：
- "OpenAI 官方 发布 GPT-5"
- "Sam Altman 采访 GPT-5"
- "OpenAI 官方博客 GPT-5 技术细节"
```

**重点信息**：
- 发布时间、版本号、主要功能
- 技术参数（模型架构、参数量、性能指标）
- 官方定位和应用场景
- 价格策略和商用政策

### 维度 2：权威媒体报道
**目标**：获取深度分析和行业视角

**搜索策略**：
```
关键词模式：
- "site:36kr.com [主题]"
- "site:huxiu.com [主题]"
- "site:geekpark.net [主题]"
- "site:techcrunch.com [主题]"

搜索示例：
- "site:36kr.com GPT-5 评测"
- "site:huxiu.com AI 写作 趋势"
- "site:geekpark.net GPT-5 vs GPT-4"
```

**重点信息**：
- 媒体评价和观点
- 与竞品的对比分析
- 行业影响和趋势判断
- 专家访谈和观点

### 维度 3：用户反馈和案例
**目标**：了解实际使用体验和真实效果

**搜索策略**：
```
关键词模式：
- "[产品名] 使用 体验"
- "[产品名] 效果 怎么样"
- "[产品名] 优缺点"
- "[产品名] 案例 实战"

搜索示例：
- "GPT-5 使用 体验"
- "AI 写作工具 效果 怎么样"
- "ChatGPT 写作 案例"
```

**重点信息**：
- 用户好评和差评
- 实际使用场景和效果
- 遇到的问题和解决方案
- 用户建议和期待

### 维度 4：数据支撑
**目标**：用数据增强文章说服力

**搜索策略**：
```
关键词模式：
- "[主题] 数据 报告"
- "[主题] 市场 规模"
- "[主题] 用户 增长"
- "[主题] 调研 问卷"

搜索示例：
- "AI 写作 市场 规模"
- "ChatGPT 用户 数据"
- "内容创作 工具 调研 报告"
```

**重点信息**：
- 市场规模和增长率
- 用户数量和增长率
- 使用频率和满意度
- 行业渗透率

### 维度 5：争议和批评
**目标**：保持文章客观性，呈现多面观点

**搜索策略**：
```
关键词模式：
- "[产品名] 问题 缺陷"
- "[产品名] 批评 质疑"
- "[产品名] 争议 事件"

搜索示例：
- "GPT-5 问题 缺陷"
- "ChatGPT 批评 质疑"
- "AI 写作 风险 挑战"
```

**重点信息**：
- 已知的问题和缺陷
- 行业内的质疑声音
- 可能的风险和挑战
- 用户担忧的问题

## 搜索工具使用

### 搜索方法
根据智能体具备的网络访问能力，采用以下搜索策略：

1. **多轮搜索**：设计 3-5 轮搜索，每轮聚焦不同维度
   - 第 1 轮：官方资料和权威媒体
   - 第 2 轮：用户反馈和案例
   - 第 3 轮：数据支撑和报告
   - 第 4 轮：争议和批评

2. **交叉验证**：对比不同来源的信息
   - 数据是否一致
   - 观点是否冲突
   - 时间是否匹配

3. **深度搜索**：针对关键信息进行深度挖掘
   - 追溯到原始来源
   - 查找更多细节
   - 验证信息的真实性

### 搜索记录
**记录格式**：
```
[信息内容]
来源：[来源名称]
发布时间：[YYYY-MM-DD]
URL：[链接]
可信度：[高/中/低]
```

**示例**：
```
GPT-5 参数量达到 10 万亿
来源：OpenAI 官方博客
发布时间：2024-03-01
URL：https://openai.com/blog/gpt-5
可信度：高
```

## 信息验证方法

### 1. 交叉验证
**方法**：对比 2-3 个不同来源的信息

**检查项**：
- 数据是否一致
- 时间是否匹配
- 观点是否冲突

**处理方式**：
- 一致：接受信息
- 不一致：进一步搜索验证
- 无法验证：标注"据称"或删除

### 2. 来源可信度评估
**评估标准**：
- 官方来源：可信度最高
- 权威媒体：可信度高
- 专业报告：可信度高
- 用户反馈：可信度中等
- 社区讨论：可信度中等偏低

**使用原则**：
- 优先使用可信度高的来源
- 低可信度来源需要交叉验证
- 无法验证的信息明确标注

### 3. 时效性检查
**检查项**：
- 信息发布时间
- 信息是否过时
- 是否有最新更新

**处理方式**：
- 最新信息（1-3 个月）：优先使用
- 中期信息（6-12 个月）：谨慎使用
- 过时信息（>1 年）：明确标注时间背景

### 4. 逻辑合理性检查
**检查项**：
- 数据是否合理（是否存在明显矛盾）
- 逻辑是否清晰
- 是否存在明显错误

**处理方式**：
- 发现逻辑问题：删除或标注
- 无法判断：进一步验证
- 确认无误：保留

## 搜索示例

### 示例 1：搜索 GPT-5 的技术细节
**第 1 轮：官方资料**
- 搜索："OpenAI 官方 GPT-5 技术细节"
- 搜索："OpenAI 博客 GPT-5 架构"
- 结果：获取参数量、架构、性能等官方信息

**第 2 轮：权威媒体**
- 搜索："site:36kr.com GPT-5 技术分析"
- 搜索："site:huxiu.com GPT-5 vs GPT-4"
- 结果：获取媒体分析、对比评价

**第 3 轮：用户反馈**
- 搜索："GPT-5 使用 体验 评价"
- 搜索："GPT-5 效果 怎么样"
- 结果：获取实际使用感受

**第 4 轮：交叉验证**
- 对比不同来源的信息
- 识别冲突点
- 进一步验证不确定的信息

### 示例 2：搜索 AI 写作工具市场数据
**第 1 轮：市场报告**
- 搜索："AI 写作 市场 规模 报告"
- 搜索："ChatGPT 用户 增长 数据"
- 结果：获取市场规模、用户数据

**第 2 轮：行业分析**
- 搜索："AI 写作 行业 趋势"
- 搜索："内容创作 工具 渗透率"
- 结果：获取行业趋势、渗透率数据

**第 3 轮：竞品对比**
- 搜索："AI 写作工具 对比 评测"
- 搜索："ChatGPT vs Claude vs 文心一言"
- 结果：获取竞品对比数据

**第 4 轮：验证**
- 对比多个报告的数据
- 验证数据的一致性
- 标注数据来源和时间

## 注意事项
- **多源验证**：至少 2-3 个来源，避免单一来源偏差
- **时效性**：优先使用最新信息，标注信息发布时间
- **可信度**：优先使用官方和权威媒体来源
- **完整性**：搜索要覆盖多个维度，避免信息遗漏
- **客观性**：搜索正反面信息，保持文章客观
- **记录来源**：每个信息都要标注来源，方便核查
