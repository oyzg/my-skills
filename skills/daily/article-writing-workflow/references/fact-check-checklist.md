## 概览
本文档提供系统性的事实核查清单，用于验证 AI 生成内容的真实性。通过多维度核查，避免 AI 幻觉问题，确保文章内容真实可信。

## 核查维度

### 1. 数据真实性核查
**核查要点**：
- [ ] 数据来源是否明确
  - 是否标注数据来源（公司财报、研究报告、官方发布）
  - 数据来源是否权威（优先级：官方 > 权威媒体 > 一般媒体 > 个人）

- [ ] 数据是否准确
  - 数据是否与原始来源一致
  - 单位、时间范围是否正确
  - 数据是否过时（核查发布时间）

- [ ] 数据是否有逻辑问题
  - 数据是否合理（是否存在明显矛盾）
  - 数据之间是否存在逻辑冲突

**示例**：
```
❌ 未标注来源：
"GPT-5 参数量达到 100 万亿，是上一代的 10 倍"
→ 需要验证：OpenAI 是否发布过这个数据？

✅ 标注来源且准确：
"根据 OpenAI 2024 年 3 月的发布信息，GPT-5 参数量达到 10 万亿，是上一代的 10 倍"
→ 可信度高：有明确来源、具体时间、对比明确
```

### 2. 事实陈述核查
**核查要点**：
- [ ] 核心事实是否有来源支撑
  - 事件是否真实发生
  - 时间、地点、人物是否准确
  - 是否存在张冠李戴

- [ ] 多源交叉验证
  - 至少查询 2-3 个不同来源
  - 对比各来源的信息是否一致
  - 记录信息差异点

- [ ] 模糊信息处理
  - 无法验证的信息：删除或明确标注"据称""传闻"
  - 过时信息：标注时间背景
  - 争议信息：多方观点都呈现

**示例**：
```
❌ 无法验证：
"OpenAI CEO Sam Altman 表示，GPT-5 将在 2024 年 6 月发布"
→ 需要验证：Sam Altman 确实说过这句话吗？具体场合？

✅ 可验证：
"根据 TechCrunch 2024 年 3 月 15 日的报道，Sam Altman 在 SXSW 大会上表示，GPT-5 将在 2024 年内发布，但未透露具体时间"
→ 可信度高：有明确来源、时间、场合、原话
```

### 3. 技术描述核查
**核查要点**：
- [ ] 技术原理是否正确
  - 技术概念是否准确
  - 技术描述是否符合业界共识
  - 是否存在技术性错误

- [ ] 技术术语使用是否准确
  - 术语含义是否正确
  - 术语使用场景是否合适
  - 是否存在术语混用

- [ ] 技术对比是否客观
  - 对比维度是否合理
  - 对比数据是否真实
  - 是否存在误导性对比

**示例**：
```
❌ 技术描述错误：
"GPT-5 采用了量子计算技术，所以速度比 GPT-4 快 100 倍"
→ 核查：GPT-5 真的使用量子计算吗？这个说法有来源吗？

✅ 技术描述准确：
"根据 OpenAI 技术博客，GPT-5 在模型架构上优化了注意力机制，推理速度比 GPT-4 提升约 50%"
→ 可信度高：有明确来源、具体技术点、合理的数据对比
```

### 4. 人物/公司信息核查
**核查要点**：
- [ ] 人物身份是否准确
  - 姓名、职位、公司是否正确
  - 是否存在人物混淆
  - 引用的话是否是本人所说

- [ ] 公司信息是否准确
  - 公司名称、成立时间、业务范围是否正确
  - 公司是否真实存在
  - 公司动态是否真实

- [ ] 人物观点是否准确
  - 观点是否被曲解
  - 语境是否完整
  - 是否存在断章取义

**示例**：
```
❌ 人物信息错误：
"百度 CEO 李彦宏表示，文心一言已经超过 GPT-4"
→ 核查：李彦宏真的说过这句话吗？具体场合？是否有录音/文字记录？

✅ 人物信息准确：
"在 2024 年 2 月的财报电话会议上，百度 CEO 李彦宏表示，文心一言在中文场景下的某些指标已经接近 GPT-4"
→ 可信度高：有明确场合、具体表述、合理对比
```

### 5. 时间/地点核查
**核查要点**：
- [ ] 时间是否准确
  - 年份、月份、日期是否正确
  - 时间顺序是否合理
  - 是否存在时间穿越

- [ ] 地点是否准确
  - 国家、城市、场馆名称是否正确
  - 事件发生在哪个地方
  - 是否存在地点混淆

**示例**：
```
❌ 时间错误：
"2023 年 12 月，OpenAI 发布了 GPT-5"
→ 核查：GPT-5 真的在 2023 年 12 月发布了吗？

✅ 时间准确：
"2024 年 3 月，OpenAI 正式发布了 GPT-5"
→ 需要标注具体发布日期和来源
```

## 核查流程

### 步骤 1：信息来源验证
对每个关键信息点执行：
1. 识别信息类型（数据、事实、技术、人物、时间）
2. 查找原始来源
3. 标注来源信息（来源名称、发布时间、URL）

### 步骤 2：多源交叉验证
1. 查找 2-3 个其他来源
2. 对比各来源信息的一致性
3. 记录差异点并进一步验证

### 步骤 3：逻辑合理性检查
1. 检查信息之间的逻辑关系
2. 识别矛盾点
3. 标注需要删除或修正的内容

### 步骤 4：标注和修正
1. 对所有关键信息标注来源
2. 删除无法验证的信息
3. 修正错误的信息
4. 标注不确定的信息

## 信息标注规范

### 1. 数据标注格式
```
标准格式：
[数据内容]（来源：[来源名称]，发布时间：[YYYY-MM-DD]）

示例：
GPT-5 参数量达到 10 万亿（来源：OpenAI 官方博客，发布时间：2024-03-01）
```

### 2. 事实陈述标注格式
```
标准格式：
[事实内容]（来源：[来源名称]，链接：[URL]，发布时间：[YYYY-MM-DD]）

示例：
Sam Altman 在 SXSW 大会上表示 GPT-5 将在 2024 年内发布（来源：TechCrunch，链接：https://techcrunch.com/...，发布时间：2024-03-15）
```

### 3. 人物观点标注格式
```
标准格式：
[人物姓名] [职位] 在 [场合] 表示："[观点]"（来源：[来源名称]，发布时间：[YYYY-MM-DD]）

示例：
Sam Altman（OpenAI CEO）在 SXSW 大会上表示："GPT-5 将在 2024 年内发布，但我们还需要解决一些问题"（来源：TechCrunch，发布时间：2024-03-15）
```

### 4. 不确定信息标注格式
```
标准格式：
据称/传闻/未经证实：[信息内容]（来源：[来源名称]，发布时间：[YYYY-MM-DD]）

示例：
据称 GPT-5 可能采用新的模型架构（来源：TechCrunch，发布时间：2024-03-10）
```

## 常见幻觉类型

### 1. 数据幻觉
**表现**：
- 编造不存在的数据
- 张冠李戴（把 A 公司的数据安在 B 公司头上）
- 数据单位错误（10 万亿写成 10 亿）

**识别方法**：
- 核查数据来源
- 对比多个来源
- 检查数据合理性

### 2. 事件幻觉
**表现**：
- 编造不存在的事件
- 混淆事件时间
- 夸大事件影响

**识别方法**：
- 查找事件报道
- 核查时间线
- 对比多个来源

### 3. 技术幻觉
**表现**：
- 编造技术细节
- 错误解释技术原理
- 夸大技术能力

**识别方法**：
- 查阅技术文档
- 对比官方说明
- 咨询专业资料

### 4. 人物/公司幻觉
**表现**：
- 编造人物言论
- 混淆人物身份
- 编造公司信息

**识别方法**：
- 核查人物身份
- 查找原始记录
- 核查公司官方信息

### 5. 引用幻觉
**表现**：
- 编造引用内容
- 错误标注来源
- 断章取义

**识别方法**：
- 查找原文
- 对比上下文
- 核查原文时间

## 注意事项
- 严格核查：所有关键信息必须验证，不能假设
- 诚实标注：无法验证的信息必须明确标注
- 及时删除：无法验证且无法标注的信息应删除
- 持续更新：信息可能过时，需要定期核查
- 交叉验证：单一来源不能作为最终依据
