# 公众号文章全自动写作工作流

## 任务目标
- 本 Skill 用于：创作 AI & 科技类公众号爆款文章
- 能力包含：风格学习、公众号复刻模仿、选题评估、网络搜索、结构化写作、自动检验并修改、配图建议/生成、文档导出
- 触发条件：用户需要创作公众号文章，特别是 AI & 科技领域

## 前置准备
- 准备参考文章：提供 2-3 篇符合目标风格的优秀文章（用于风格学习）
- 明确主题方向：准备文章的核心主题或选题范围
- 依赖包：如需导出 Word/PDF 格式并支持水印去除，需安装以下依赖
  ```
  pip install python-docx reportlab Pillow
  ```

## 操作步骤

### 步骤一：风格学习与公众号复刻模仿
1. **分析参考文章**
   - 读取用户提供的参考文章（2-5 篇）
   - 基于 [references/writing-style-guide.md](references/writing-style-guide.md) 中的风格分析维度，提取以下特征：
     * 句式特点（长短句比例、句型结构）
     * 词汇选择（专业术语 vs 通俗表达、高频词汇）
     * 修辞手法（比喻、对比、设问的使用频率）
     * 语气态度（严肃专业 vs 轻松幽默）
     * 段落节奏（段落长度、过渡方式、标点使用）

2. **公众号深度复刻模仿**
   - 如果是公众号文章复刻需求，执行以下深度分析：
     * 标题风格（痛点型、悬念型、数据型、故事型等）
     * 开头套路（黄金 3 秒的常用模式）
     * 结尾结构（总结 + 行动召唤 + 情绪升华）
     * 信息密度（每段话的信息量和节奏）
     * 人设特征（作者的个人风格、常用表达）
     * 情绪曲线（文章的情绪起伏和共鸣点）

3. **生成风格报告**
   - 输出风格特征总结（300-500 字）
   - 列出需要模仿的关键要素（3-5 条）
   - 识别目标读者群体
   - 如果是公众号复刻，提供详细的模仿指南

### 步骤二：选题讨论
1. **主题评估**
   - 用户提出主题或方向
   - 智能体从以下维度评估选题潜力：
     * 时效性（是否是当前热点）
     * 痛点价值（解决什么问题）
     * 差异化（与已有文章的区别）
     * 传播潜力（是否具备爆点）

2. **选题优化建议**
   - 提供角度优化建议
   - 建议具体的切入点和视角
   - 预估文章影响力

### 步骤三：网络搜索与信息验证
1. **系统化网络搜索**
   - 参考 [references/network-search-guide.md](references/network-search-guide.md)，执行多维度搜索
   - 搜索策略：
     * 第 1 轮：官方资料和权威媒体（官网、官方博客、36氪、虎嗅、极客公园）
     * 第 2 轮：用户反馈和案例（知乎、小红书、微博）
     * 第 3 轮：数据支撑（艾瑞咨询、易观分析、IDC 报告）
     * 第 4 轮：争议和批评（多方观点，保持客观）

2. **关键词设计**
   - 从宽泛到精准，逐步细化：
     * 第 1 轮：核心关键词（宽泛）："GPT-5"、"OpenAI"、"AI 写作"
     * 第 2 轮：组合关键词（增加限定）："GPT-5 发布时间"、"AI 写作 工具 对比"
     * 第 3 轮：精确关键词（目标明确）："GPT-5 参数量 多少"、"OpenAI 2024年3月 发布会"

3. **信息来源验证**
   - 优先级顺序：官方来源 > 权威媒体 > 专业报告 > 用户反馈 > 社区讨论
   - 记录每个信息的来源、发布时间、URL、可信度
   - 对比 2-3 个不同来源的信息，识别冲突点
   - **深度要求**：每个关键观点至少收集 2-3 个不同来源的详细信息，包括数据、案例、专家观点等

4. **交叉验证**
   - 对比多个来源的信息
   - 识别冲突点并进一步验证
   - 记录信息来源和可信度

5. **信息整理**
   - 提取关键事实和数据
   - 标注信息来源（格式：[内容] 来源：[来源名称]，发布时间：[YYYY-MM-DD]）
   - 整理成信息结构树
   - **收集要求**：
     * 每个核心观点至少收集 3-5 个支撑数据或案例
     * 收集历史背景、发展脉络、现状分析
     * 收集专家观点、行业报告、用户反馈
     * 收集对比分析、优势劣势、适用场景
     * 收集未来趋势、挑战与机遇

6. **事实核查**
   - 基于 [references/fact-check-checklist.md](references/fact-check-checklist.md) 逐项核查
   - 记录所有不确定的信息
   - 删除无法验证的信息或明确标注

### 步骤四：结构化写作
1. **选择结构模板**
   - 根据 [references/viral-article-structures.md](references/viral-article-structures.md) 选择合适的结构
   - 常用结构：
     * 问题-解决方案型
     * 趋势分析型
     * 技术解读型
     * 案例故事型

2. **生成黄金开头**
   - 使用 [assets/title-templates.txt](assets/title-templates.txt) 和 [assets/opening-templates.txt](assets/opening-templates.txt)
   - **核心原则**：前 3 秒必须抓住读者注意力，制造"想继续看下去"的欲望
   - **开头类型选择**（根据主题选择最适合的）：
     * 痛点型：直接戳中读者痛点，引发共鸣
     * 悬念型：制造悬念，引发好奇
     * 数据冲击型：用惊人数据吸引注意
     * 故事型：用故事开头，引发情感共鸣
     * 对比反差型：用强烈对比制造冲击力
     * 提问型：用问题引发思考
   - **开头结构**（黄金3秒 + 利益点 + 过渡）：
     * 第 1 句（黄金3秒）：必须包含冲突/悬念/痛点/数据冲击，不要铺垫
       - ❌ 错误："最近，OpenAI 发布了 GPT-5，这是一个重要的里程碑..."
       - ✅ 正确："GPT-5 发布后，90% 的写作者慌了"
     * 第 2 句（利益点）：给出读者能得到什么
       - "学会这 3 招，效率提升 10 倍"
     * 第 3-4 句（过渡）：解释原因，引出正文
       - "因为 GPT-5 不只是写作工具，它正在改变整个内容行业..."
   - **开头痛点清单**：
     * 是否有惊人的数据或事实？
     * 是否戳中了读者的痛点？
     * 是否制造了悬念或好奇心？
     * 读者能得到什么具体利益？
     * 是否与读者有直接关联？
   - **常见错误**（必须避免）：
     * 铺垫太长，开头超过 100 字还没进入主题
     * 过于平淡，没有吸引力
     * 无利益点，读者不知道能得到什么
     * 与读者无关，无法引发共鸣
   - **长度控制**：开头控制在 100-200 字左右

3. **填充内容框架**
   - 基于搜索到的信息
   - 按照结构模板填充
   * 开头：黄金 3 秒 + 利益点前置
   * 中段：逻辑递进 + 信息密度
   * 结尾：行动召唤 + 情绪共鸣
   - **内容充实度要求**：
     * 每个观点至少展开 3-5 句话，提供充分的解释和说明
     * 每个关键点至少包含 2-3 个具体案例或数据支撑
     * 提供背景信息：历史发展、技术原理、市场现状等
     * 提供深度分析：影响分析、对比分析、优劣势分析
     * 提供实际应用：使用场景、操作指南、最佳实践
     * 提供未来展望：发展趋势、挑战机遇、行动建议
     * 避免空洞表述，确保每个段落都有实质性内容

4. **应用风格特征**
   - 应用步骤一提取的风格要素
   * 模仿句式和词汇选择
   * 保持一致的语气态度
   * 使用相似的修辞手法

5. **生成初稿**
   - 输出完整文章初稿
   - **质量优先原则**：根据主题需要自然展开，确保内容充实完整，不人为限制字数
   - **内容充实标准**：
     * 每个核心观点都有充分的论证和说明
     * 每个技术概念都有清晰的解释和示例
     * 每个案例都有详细的背景、过程、结果
     * 每个数据都有来源、背景、意义分析
     * 每个建议都有具体的实施步骤和注意事项
   - 确保逻辑清晰、层次分明
   - 预留信息来源标注位置
   - 目标：根据主题复杂度，通常在 5000-8000 字，但以内容完整为第一优先级

### 步骤五：审校优化
1. **事实核查**
   - 使用 [references/fact-check-checklist.md](references/fact-check-checklist.md) 逐项检查
   - 验证所有数据和引用
   - 补充或删除无法验证的内容
   - **原则**：删除时仅删除明显错误或无法验证的内容，保留有价值的信息

2. **去 AI 感优化**
   - 使用 [references/editorial-review-checklist.md](references/editorial-review-checklist.md)
   * 消除机械句式（避免过度使用"首先、其次、综上所述"）
   * 打破三段式结构（增加过渡句、长短句交替）
   * 注入情感和观点（加入个人见解、情绪表达）
   * 使用口语化表达（适度使用反问、感叹、省略）
   * **重要**：优化表达时不要删减有价值的内容，通过改写、展开、举例等方式增强表达效果

3. **爆款元素强化**
   * 检查标题是否有吸引力
   * 优化开头是否在前 3 秒抓住注意力
   * 强化利益点是否清晰
   * 增强情绪共鸣点
   * **补充要求**：如果某个部分不够吸引人，通过增加案例、数据、对比等方式强化，而不是简单删除

### 步骤六：自动检验并修改文章
1. **第一轮：错别字和基础错误检查**
   - 参考 [references/three-round-review-guide.md](references/three-round-review-guide.md) 第一轮
   * 逐字检查，重点关注：
     - 错别字（的/地/得、形近字、音近字）
     - 标点符号（逗号、句号、引号是否正确）
     - 语法错误（主谓宾是否齐全、时态是否一致）
     - 格式错误（数字、时间、引用格式是否统一）
     - 逻辑矛盾（数据前后是否一致、观点是否冲突）

2. **第一轮修改**
   - 记录发现的问题（格式：[位置] 问题：具体问题）
   - 逐项修改：
     * 找到问题的具体位置
     * 删除错误的字/词/标点
     * 插入正确的内容
     - 示例：
       ```
       [第 3 段第 2 句] 问题："的的"重复
       修改前："这个功能很好用，你会喜欢的的。"
       修改后："这个功能很好用，你会喜欢的。"
       ```
   - 修改后重读，确保修改正确

3. **第二轮：表达优化（重点开头和结尾）**
   - 参考 [references/three-round-review-guide.md](references/three-round-review-guide.md) 第二轮
   * 开头优化：
     - 第 1 句检查：是否抓住注意力（冲突/悬念/痛点/数据）
     - 利益点检查：是否前置利益点（告诉读者能得到什么）
     - 悬念检查：是否制造继续阅读的欲望
     - 长度检查：长度是否合适（50-100 字）
   * 结尾优化：
     - 总结检查：是否有总结提炼（3 条核心要点）
     - 行动召唤检查：是否有行动召唤（告诉读者下一步做什么）
     - 情绪升华检查：是否有情绪升华（共鸣、希望、激励）
     - 长度检查：长度是否合适（100-200 字）
   * 中段优化：
     - 节奏检查：段落节奏是否有变化（短段落、问句、感叹句）
     - 逻辑检查：逻辑是否清晰（段落过渡是否自然）
     - 信息密度检查：信息密度是否合理（每段话都有价值）

4. **第二轮修改**
   - 记录发现的优化点（格式：[位置] 问题：具体问题）
   - 逐项优化：
     * 找到需要优化的具体位置
     - 分析原因（为什么需要优化）
     - 重写内容
     - 示例：
       ```
       [开头第 1 句] 问题：太平淡，没有抓住注意力
       修改前："今天我们来聊聊 GPT-5。"
       修改后："你每天都在浪费 2 小时，而且你自己都不知道。"
       原因：第 1 句直接抛出痛点，比平淡开场更有冲击力
       ```
     * 示例 2：
       ```
       [结尾] 问题：无总结，无行动召唤
       修改前："以上就是 GPT-5 的介绍，希望能对你有帮助。"
       修改后："总结一下，GPT-5 的突破主要体现在三个方面：参数量翻倍、推理能力提升、多模态能力增强。今天就开始用 GPT-5 吧，评论区告诉我你的使用体验！"
       原因：明确总结要点，给出行动召唤，增强互动
       ```
   - 修改后重读，确保优化有效

5. **第三轮：口语化检查**
   - 参考 [references/three-round-review-guide.md](references/three-round-review-guide.md) 第三轮
   * 机械句式检查：
     - 检查"首先、其次、再次、最后"
     - 检查"综上所述、总而言之"
     - 检查"一方面、另一方面"
   * 口语化表达检查：
     - 检查"因此、所以"是否可以用"这就意味着、也就是说"
     - 检查是否可以使用反问句、设问句
     - 检查是否可以使用感叹词（"真的、太、居然"）
   * 情绪和观点检查：
     - 检查是否有明确的观点和态度
     - 检查是否可以使用"我觉得、说实话"
     - 检查是否可以表达反对、支持、惊喜、担忧等情绪
   * 三段式结构检查：
     - 检查段落长度是否过于平均
     - 检查是否可以用短段落、问句、感叹句打破节奏
   * **内容充实度检查**（新增）：
     - 检查每个关键观点是否充分展开（至少 3-5 句话）
     - 检查是否有足够的案例和数据支撑（至少 2-3 个）
     - 检查过于简短的部分（少于 100 字的段落），考虑补充
     - 检查是否有空洞表述，需要添加具体内容
     - 检查是否有可以深化的观点，增加分析维度
     - **重要**：不要为了简洁而删除有价值的内容，确保内容充实

6. **第三轮修改**
   - 记录发现的口语化问题（格式：[位置] 问题：具体问题）
   - 逐项优化：
     * 找到需要优化的具体位置
     - 分析原因（为什么需要口语化）
     - 重写内容
     - 示例：
       ```
       [第 2 段] 问题：机械句式"首先、其次"
       修改前："首先，GPT-5 的发布具有重要意义。其次，它将改变内容创作方式。"
       修改后："GPT-5 的发布意义重大，它正在重塑内容创作。更有意思的是，很多企业已经开始提前布局了。"
       原因：删除机械句式，用更自然的表达，增加"更有意思的是"增加节奏感
       ```
     * 示例 2：
       ```
       [第 3 段] 问题：过于正式，无情绪
       修改前："该模型的表现令人满意。"
       修改后："说实话，这个模型的表现还挺让人惊喜的。"
       原因：加入"说实话"和"还"，增加口语化和情绪
       ```
     * 示例 3（内容充实度）：
       ```
       [第 5 段] 问题：内容过于简短，缺少具体说明
       修改前："GPT-5 的参数量很大。"
       修改后："GPT-5 的参数量达到了惊人的 10 万亿级别，是 GPT-4 的 10 倍以上。这意味着它在理解复杂概念、推理逻辑、多轮对话方面的能力会有质的飞跃。根据 OpenAI 官方披露，新模型在数学推理、代码生成、跨模态理解等任务的准确率提升了 30-50%。"
       原因：补充具体数据、对比分析、能力提升说明，确保内容充实
       ```
   - 修改后重读，确保口语化自然
   - **重要**：在优化表达时，不要删除有价值的内容，而是通过补充、展开、举例等方式增强内容

7. **最终质量检查**
   - 检查清单：
     * 无错别字、标点错误、语法错误
     * 开头第 1 句抓人（冲突/悬念/痛点/数据）
     * 开头有利益点（读者能得到什么）
     * 开头有悬念（想继续看下去）
     * 结尾有总结（3 条核心要点）
     * 结尾有行动召唤（告诉读者下一步做什么）
     * 结尾有情绪升华（共鸣、希望、激励）
     * 无机械句式（首先、其次、最后）
     * 有口语化表达（所以、我觉得、说实话）
     * 有短段落（1-2 句）
     * 有情绪和观点（真的、太、居然）
     * **内容充实度**（新增）：
       - 每个关键观点都有充分展开（不少于 3 句话）
       - 有足够的案例和数据支撑（不少于 2-3 个）
       - 无空洞表述，每个段落都有实质性内容
       - 有深度分析（影响、对比、趋势等）
       - 有实际应用指导（场景、步骤、建议）

8. **输出修改日志**
   - 输出 3 轮审核发现的所有问题和修改内容
   - 格式：
     ```
     第一轮：错别字和基础错误检查
     - [位置] 问题：xxx
       修改前：xxx
       修改后：xxx

     第二轮：表达优化
     - [位置] 问题：xxx
       修改前：xxx
       修改后：xxx

     第三轮：口语化检查
     - [位置] 问题：xxx
       修改前：xxx
       修改后：xxx
     ```
   - 修改前后对比，确保修改有效
   - 输出最终成稿

### 步骤七：配图搜索与插入
1. **搜索文章对应的图片**
   - 识别文章的主题、关键词、核心观点
   - 根据配图描述搜索或生成对应图片
   - 搜索策略：
     * 使用关键词 + "图片"、"官方"、"高清"
     * 优先搜索官方图片（公司官网、官方博客、权威媒体）
     * 如果没有官方图片，使用智能体图像生成能力
   - 配图要求：
     * 横板格式（16:9 或类似比例）
     * 宽度建议：900px 或 1080px
     * 风格统一（配色、风格元素、情感氛围）

2. **裁剪四周（去除水印）**
   - 下载图片并保存到 `./images/` 目录
   - 文件命名格式：`image_01.jpg`、`image_02.jpg` 等
   - **自动裁剪四周**：
     * 调用 `scripts/export_document.py` 的水印去除功能
     * 自动裁剪图片四个角落（去除水印）
     * 裁剪比例：每侧约 5-8%
     * 保存裁剪后的图片

3. **插入到文章中**
   - 在文章中标记配图插入位置
   - 标记格式：`[IMAGE: 图片描述]`
   - 示例：
     ```
     # 文章标题

     [IMAGE: 封面图：GPT-5 概念图]

     文章开头...

     ## 第一部分

     [IMAGE: 插图：AI 写作场景]

     文章内容...
     ```
   - 导出文档时自动将图片插入到标记位置

### 步骤八：生成可下载文档（支持图片插入）
1. **准备文章内容**
   - 整理最终成稿的文章内容
   - 确保格式正确（标题、段落、引用）
   - 确保配图标记正确（`[IMAGE: 图片描述]`）
   - 确保配图文件已保存到 `./images/` 目录

2. **选择导出格式**
   - 调用 `scripts/export_document.py` 脚本
   - 支持的格式：
     * Markdown (.md)：纯文本格式，支持图片插入（使用相对路径）（默认格式）
     * Word (.docx)：可编辑的文档格式，支持图片插入
     * PDF：不可编辑的文档格式，支持图片插入
     * All：同时生成所有格式

3. **生成文档（支持图片插入与自动裁剪四周）**
   - 执行导出脚本
   - 参数：
     * `--content`: 文章内容（包含 `[IMAGE: 图片描述]` 标记）
     * `--format`: 输出格式（md/docx/pdf/all，默认为 md）
     * `--filename`: 输出文件名
     * `--images-dir`: 图片目录（默认 `./images/`）
     * `--crop-borders`: 是否自动裁剪图片四周（默认 true）
   - 脚本功能：
     * **格式检查和修复**（自动执行）：
       - 检查并修复加粗标记不匹配（奇数个 `**`）
       - 检查并修复连续加粗（`**text1****text2**` → `**text1** **text2**`）
       - 检查并删除空加粗（`******`）
       - 检测并警告嵌套加粗（`**text **inner** text**`）
       - 检测并警告斜体标记不匹配
       - 检测并警告代码标记不匹配
     * **排版优化**（Word 格式）：
       - 段落间距：段前段后 6pt，1.5 倍行距
       - 标题间距：段前 12pt，段后 6pt，1.5 倍行距
       - 列表格式：优化列表项间距和缩进
       - 统一字体：宋体 12pt
     * 解析文章中的 `[IMAGE: 图片描述]` 标记
     * 将图片插入到对应位置
     * **自动裁剪四周**（默认启用）：
       - 自动裁剪图片四个角落（去除水印）
       - 裁剪比例：每侧约 5-8%
       - 保留主体内容，去除边缘水印
     * **Markdown 格式转换**（Word 格式）：
       - 标题：`#`、`##`、`###` → Word 标题格式
       - 表格：`| 列1 | 列2 |` → Word 表格格式（商务对比风格）
         - 表头：浅灰色背景（#E0E0E0）、微软雅黑字体 10.5pt、加粗、左对齐
         - 边框：0.5pt 黑色实线边框（所有单元格）
         - 内容：白色背景、微软雅黑字体 10.5pt、左对齐
         - 内边距：上下 3pt、左右 6pt（适中留白）
         - 布局：自动调整列宽，规整紧凑
       - 无序列表：`- ` 或 `* ` → Word 列表格式
       - 有序列表：`1. `、`2. ` → Word 编号列表格式
       - 引用：`> ` → Word 引用样式（灰色斜体、左缩进）
       - 分割线：`---` 或 `***` → Word 分割线
       - 行内加粗：`**text**` → Word 加粗格式
       - 行内斜体：`*text*` → Word 斜体格式
       - 行内代码：`` `text` `` → Word 等宽字体
       - 行内删除线：`~~text~~` → Word 删除线格式
     * Markdown：使用 `

![图片描述](./images/image_01.jpg)
` 格式
     * Word：使用 `docx.shared.InlineImage` 插入图片，自动转换 Markdown 格式为 Word 格式，并进行格式检查和排版优化
     * PDF：使用 `reportlab.platypus.Image` 插入图片


4. **验证文档**
   - 检查生成的文档是否完整
   - 检查格式是否正确
   - 检查图片是否正确插入到对应位置
   - 检查图片四周是否已裁剪（去除水印）
   - 确认可正常打开和编辑

5. **输出可下载链接**
   - 提供文档下载路径
   - 说明文档格式和用途
   - 提供使用建议
